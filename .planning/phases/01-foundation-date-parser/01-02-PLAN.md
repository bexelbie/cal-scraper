    ## DATE-02: Single day + hour.minutes ("8/4/2026, 16.30 H")
    - parse_date("● 8/4/2026, 16.30 H") → ParsedDate(dtstart=datetime(2026,4,8,16,30,tzinfo=PRAGUE_TZ), dtend=datetime(2026,4,8,18,30,tzinfo=PRAGUE_TZ), all_day=False)
    - Uses dot (.) for minutes, NOT colon (:)
    - dtend is dtstart + 2 hours (D-01)

    ## DATE-03: Single day, no time — all-day event ("23/5/2026")
    - parse_date("● 23/5/2026") → ParsedDate(dtstart=date(2026,5,23), dtend=date(2026,5,24), all_day=True)
    - dtstart is a date object (not datetime)
    - dtend is dtstart + 1 day (exclusive end for iCal all-day events)

    ## DATE-04: Multi-day range, no time ("7/7 – 11/7/2026")
    - parse_date("● 7/7 – 11/7/2026") → ParsedDate(dtstart=date(2026,7,7), dtend=date(2026,7,12), all_day=True)
    - En-dash U+2013 (–) NOT hyphen U+002D (-)
    - Year inferred from end date → applied to start date
    - dtend = last day + 1 (exclusive, per D-02)
    - all_day=True (D-02: single spanning all-day event)

    ## DATE-05: Multi-day + time range ("27/7 – 31/7/2026, 9–16 H")
    - parse_date("● 27/7 – 31/7/2026, 9–16 H") → ParsedDate(dtstart=datetime(2026,7,27,9,0,tzinfo=PRAGUE_TZ), dtend=datetime(2026,7,31,16,0,tzinfo=PRAGUE_TZ), all_day=False)
    - En-dash in BOTH date range AND time range
    - D-03: DTSTART at first day's start time, DTEND at last day's end time
    - parse_date("● 13/7 – 17/7/2026, 9–16 H") → similar, different dates

    ## DATE-06: D/M/Y European date order
    - All patterns extract day FIRST, month SECOND — verified by: 31/3 = March 31 (not day 3 of month 31)
    - Single digit days/months accepted: 8/4 = April 8, 7/7 = July 7

    ## Multiple time slots (bonus format: "24/5/2026, 15 H / 16 H / 17 H")
    - parse_date("● 24/5/2026, 15 H / 16 H / 17 H") → uses first time slot: dtstart=datetime(2026,5,24,15,0,tzinfo=PRAGUE_TZ)
    - Must match BEFORE single_h pattern (/ after H prevents single_h's $ anchor)

    ## D-04: Error handling — warn and skip
    - parse_date("not a date") → returns None
    - parse_date("") → returns None
    - parse_date("some random text 2026") → returns None
    - Each None return logs a warning via logging.warning with the raw text

    - Bullet prefix "●" (with or without trailing space) stripped before pattern matching
    - Leading/trailing whitespace stripped

  <implementation>
    Implement `cal_scraper/date_parser.py` with these exact components:

    1. **Imports:** Import ParsedDate, PRAGUE_TZ, DEFAULT_DURATION from cal_scraper.models. Import re, logging, datetime, date, timedelta from stdlib.

    2. **Logger:** `logger = logging.getLogger(__name__)`

    3. **Preprocessing function `_clean(raw: str) -> str`:**
       Strip leading/trailing whitespace, then strip `●` prefix (with optional whitespace after it) using `re.sub(r'^●\s*', '', text)`.

    4. **Helper `_make_dt(day, month, year, hour, minute=0) -> datetime`:**
       Create timezone-aware datetime: `datetime(year, month, day, hour, minute, tzinfo=PRAGUE_TZ)`.

    5. **Helper `_make_date(day, month, year) -> date`:**
       Create date: `date(year, month, day)`.

    6. **Regex patterns — ORDERED from most specific to least specific:**
       Each pattern is a tuple of (compiled_regex, handler_function_reference).

       Order (CRITICAL — do NOT change):
       a. Multi-day + time range: `(\d{1,2})/(\d{1,2})\s*\u2013\s*(\d{1,2})/(\d{1,2})/(\d{4}),\s*(\d{1,2})\u2013(\d{1,2})\s*H`
       b. Multi-day no time: `(\d{1,2})/(\d{1,2})\s*\u2013\s*(\d{1,2})/(\d{1,2})/(\d{4})$`
       c. Single day + hour.minutes: `(\d{1,2})/(\d{1,2})/(\d{4}),\s*(\d{1,2})\.(\d{2})\s*H`
       d. Multiple time slots: `(\d{1,2})/(\d{1,2})/(\d{4}),\s*(\d{1,2})\s*H\s*/`
       e. Single day + hour: `(\d{1,2})/(\d{1,2})/(\d{4}),\s*(\d{1,2})\s*H$`
       f. Date only: `^(\d{1,2})/(\d{1,2})/(\d{4})$`

       Use `\u2013` (en-dash) explicitly — NOT `-` (ASCII hyphen). This is CRITICAL per Pitfall 2.
       Use `$` end-anchor on patterns e and f to prevent partial matches (Pitfall 4).

    7. **Handler functions** (one per pattern):
       - `_parse_multi_day_time(m, raw)` → D-03: DTSTART at first day + start hour, DTEND at last day + end hour. Year from end date.
       - `_parse_multi_day(m, raw)` → D-02: All-day spanning. dtend = last_day + timedelta(days=1). Year from end date.
       - `_parse_single_hm(m, raw)` → dtstart with minutes from dot-separated value, dtend = dtstart + DEFAULT_DURATION.
       - `_parse_multi_time(m, raw)` → Uses first time slot only. dtend = dtstart + DEFAULT_DURATION.
       - `_parse_single_h(m, raw)` → D-01: dtstart at whole hour, dtend = dtstart + DEFAULT_DURATION.
       - `_parse_date_only(m, raw)` → All-day. dtstart = date, dtend = date + timedelta(days=1).

    8. **Main function `parse_date(raw_text: str) -> ParsedDate | None`:**
       Clean input via `_clean()`. Try each pattern in order. On first match, call handler, return result. If no match, log warning with raw_text and return None (D-04).

    **Anti-patterns to AVOID:**
    - Do NOT use `-` (hyphen) in regex ranges — use `\u2013` (en-dash)
    - Do NOT use `:` for minutes — use `.` (dot)
    - Do NOT create naive datetimes — always pass `tzinfo=PRAGUE_TZ`
    - Do NOT redefine PRAGUE_TZ or DEFAULT_DURATION — import from models
    - Do NOT use globals() for handler dispatch — use direct function references in the pattern list
  </implementation>
